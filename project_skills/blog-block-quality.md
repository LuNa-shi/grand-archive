# Blog block quality skill

Purpose:
Define what a strong produced note / generated block looks like for this archive.

## Desired style
- concise
- technically grounded
- readable fast
- story told through concrete revealed details

## Required structure
1. what problem the source is solving
2. why that problem matters
3. what the method/system actually does
4. what the main figure reveals
5. what the experiments/demos truly validate

## Figure rule
- do not use a random decorative image
- identify the real main figure
- prefer architecture / pipeline / strongest explanatory figure
- keep the image in raw if it is a static source artifact
- embed it into produced via local link

## Writing rule
- avoid vague big-picture filler
- avoid inflated hype language
- prefer concrete system details
- distinguish claims from evidence
- say what is solid and what remains uncertain

## Success condition
The user should be able to glance at the note and quickly know:
- what this source is
- what is technically new
- whether it is actually important
